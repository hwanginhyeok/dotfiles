# HIH 3-WAY HARD REVIEW (다중 라운드 반복 리뷰)

## 개요

3개 AI 모델(Claude, Codex, GLM)이 여러 라운드에 걸쳐 코드를 리뷰하고,
각 라운드 결과를 다음 라운드에 피드백으로 제공하여 **최종 합의(consensus)** 를 도출하는
고강도 리뷰 스킬.

단발성 3-way 리뷰(hih-3way)와 달리, 모델들이 서로의 의견을 교차 검토하며
수렴(convergence) 과정을 거친다. 최종적으로 합의된 이슈와 미합의 이슈를 분리하여
PM이 판정할 수 있도록 한다.

---

## 라운드 구조

```
Round 1: 각 모델 독립 리뷰 (blinded — 다른 모델 결과 모름)
  → 결과 수집 → 비교 테이블 생성

Round 2: 각 모델이 Round 1 결과를 보고 재리뷰
  → 다른 모델이 찾은 이슈에 동의/반박/추가
  → 수렴도(coverage) 측정

Round 3 (필요시): 미합의 이슈만 집중 재리뷰
  → 최종 판정
```

### 종료 조건
- **100% 합의** → 즉시 종료 (어느 라운드에서든)
- **2라운드 후 수렴도 ≥ 80%** → 종료
- **최대 3라운드** → 강제 종료 + 미합의 항목 PM 판정

---

## 사전 준비

### 1. 리뷰 대상 확인

사용자로부터 리뷰 대상(파일 경로, 디렉토리, PR, diff 등)을 받는다.

```
리뷰 대상: $TARGET
```

### 2. 작업 디렉토리 초기화

```bash
mkdir -p /tmp/hih_3way_hard
rm -f /tmp/hih_3way_hard/*.txt  # 이전 결과 초기화
```

### 3. 모델 정의

| 역할    | 모델     | 호출 방식                    |
|---------|----------|------------------------------|
| 리뷰어 A | Claude   | `claude` CLI 또는 API        |
| 리뷰어 B | Codex    | `codex` CLI 또는 API         |
| 리뷰어 C | GLM      | `glm` CLI 또는 API           |

> 각 모델 호출은 병렬 실행을 원칙으로 한다 (한 라운드 내 3개 호출은 동시).

---

## 실행 순서

### Round 1 — 독립 리뷰 (Blinded)

각 모델에게 **동일한 프롬프트**로 리뷰를 요청한다.
다른 모델의 존재나 결과를 알리지 않는다 (blinded).

**프롬프트 템플릿:**

```
다음 코드를 리뷰하라. 목표: 버그, 보안 문제, 성능 문제, 설계 결함, 가독성 문제를 찾아라.

[리뷰 대상 코드]
$CODE

출력 형식 (반드시 준수):
각 이슈를 다음 형식으로 보고하라.

ISSUE-N: [제목]
  심각도: CRITICAL | HIGH | MEDIUM | LOW | INFO
  위치: [파일:라인 또는 함수/클래스명]
  설명: [문제 설명]
  권고: [수정 방법]
```

**실행:**
- Claude → `/tmp/hih_3way_hard/r1_claude.txt`
- Codex → `/tmp/hih_3way_hard/r1_codex.txt`
- GLM → `/tmp/hih_3way_hard/r1_glm.txt`

**Round 1 비교 테이블 생성:**

세 결과를 읽어 이슈를 정규화(merge)한다.
유사 이슈는 동일 ID로 그룹핑한다.

```
## Round 1 비교
| 이슈 ID | 설명            | Claude | Codex | GLM | 합의 수 |
|---------|-----------------|--------|-------|-----|---------|
| I-01    | SQL 인젝션      | O      | O     | O   | 3/3     |
| I-02    | N+1 쿼리        | O      | X     | O   | 2/3     |
| I-03    | 네이밍 컨벤션   | X      | O     | X   | 1/3     |
```

합의 수 계산:
- 3/3: 전원 동의 → 합의 완료
- 2/3: 과반 동의 → 후보 합의
- 1/3: 단독 의견 → 재검토 필요

**수렴도 = (3/3 또는 2/3 합의 이슈 수) / (전체 고유 이슈 수) × 100**

---

### Round 2 — 교차 리뷰 (Cross-Review)

각 모델에게 **Round 1 전체 비교 테이블**을 제공하고 재리뷰를 요청한다.
다른 모델이 찾은 이슈에 대해 동의/반박/추가 의견을 낸다.

**프롬프트 템플릿:**

```
3명의 리뷰어가 코드를 리뷰한 Round 1 결과다.
당신의 의견을 다시 검토하라.

[Round 1 비교 테이블]
$R1_TABLE

[당신의 Round 1 리뷰]
$MY_R1_REVIEW

지시사항:
1. 다른 리뷰어가 찾았지만 당신이 놓친 이슈에 대해:
   - 동의 (AGREE): 해당 이슈가 타당하다
   - 반박 (DISAGREE): 해당 이슈가 오답 또는 과잉이다 (이유 명시)
   - 보류 (UNSURE): 추가 정보가 필요하다
2. 다른 리뷰어가 반박한 당신의 이슈에 대해 재확인하라:
   - 철회 (WITHDRAW): 내 이슈가 틀렸음을 인정
   - 유지 (MAINTAIN): 여전히 유효함 (추가 근거 제시)
3. 새로 발견한 이슈가 있으면 추가하라.

출력 형식:
ISSUE-I-XX: [이슈]
  내 Round 1 입장: [보고/미보고]
  재검토 입장: AGREE | DISAGREE | WITHDRAW | MAINTAIN | UNSURE | NEW
  이유: [설명]
```

**실행:**
- Claude → `/tmp/hih_3way_hard/r2_claude.txt`
- Codex → `/tmp/hih_3way_hard/r2_codex.txt`
- GLM → `/tmp/hih_3way_hard/r2_glm.txt`

**Round 2 분석:**

각 이슈별 최종 입장을 정리한다.

```
## Round 2 결과
| 이슈 ID | 설명        | Claude      | Codex       | GLM         | 최종 상태      |
|---------|-------------|-------------|-------------|-------------|----------------|
| I-01    | SQL 인젝션  | MAINTAIN    | AGREE       | AGREE       | 합의 (3/3)     |
| I-02    | N+1 쿼리    | MAINTAIN    | DISAGREE    | MAINTAIN    | 부분합의 (2/3) |
| I-03    | 네이밍      | AGREE       | WITHDRAW    | AGREE       | 부분합의 (2/3) |
```

**수렴도 재계산:**
- 합의 완료 = 3/3 동의 (또는 WITHDRAW로 이슈 제거)
- 수렴도 = (합의 완료 이슈 수) / (유효 이슈 수) × 100

**종료 판정:**
- 수렴도 = 100% → 즉시 최종 보고서 생성
- 수렴도 ≥ 80% → 최종 보고서 생성 (미합의 항목은 PM 판정 표시)
- 수렴도 < 80% → Round 3 진행

---

### Round 3 — 집중 재리뷰 (Focus Review)

미합의 이슈(2/3 또는 1/3)만 추출하여 집중 리뷰.

**프롬프트 템플릿:**

```
2라운드까지 진행했지만 합의에 도달하지 못한 이슈들이다.
각 이슈에 대해 최종 판정을 내려라.

[미합의 이슈 목록 + Round 2 입장 요약]
$UNRESOLVED_ISSUES

지시사항:
1. 각 이슈당 최종 결론을 하나로 좁혀라:
   - CONFIRMED: 실제 문제, 수정 필요
   - REJECTED: 문제 아님, 무시
   - DEFER: 추가 조사 필요 (PM 판정)
2. 근거를 간결하게 제시하라 (2-3문장).
3. 새 이슈 추가는 금지. 주어진 이슈에만 판정하라.

출력:
ISSUE-I-XX: [이슈]
  최종 판정: CONFIRMED | REJECTED | DEFER
  근거: [설명]
```

**실행:**
- Claude → `/tmp/hih_3way_hard/r3_claude.txt`
- Codex → `/tmp/hih_3way_hard/r3_codex.txt`
- GLM → `/tmp/hih_3way_hard/r3_glm.txt`

**최종 합의 산출:**

각 미합의 이슈에 대해 3개 모델의 Round 3 판정을 집계.

```
CONFIRMED 2/3 이상 → 수정 권고
REJECTED  2/3 이상 → 기각
그 외      → PM 판정 (DEFER)
```

## 최종 보고서 생성

모든 라운드 완료 후 (또는 조기 종료 시) 최종 보고서를 작성한다.

**출력 파일:** `/tmp/hih_3way_hard/final_report.md`

**형식:**

```markdown
## 3-WAY HARD REVIEW — FINAL REPORT

- 리뷰 대상: $TARGET
- 라운드 수: N (최대 3)
- 총 소요: ~N분
- 수렴도: X% (최종)
- 리뷰어: Claude, Codex, GLM

### 합의된 이슈 — 수정 권고 (CONFIRMED)

| #  | 이슈 ID | 이슈          | 심각도   | 합의   | 권고 수정         |
|----|---------|---------------|----------|--------|-------------------|
| 1  | I-01    | SQL 인젝션    | CRITICAL | 3/3    | 파라미터화 쿼리   |
| 2  | I-02    | N+1 쿼리      | HIGH     | 2/3    | 배치 로딩         |

### 기각된 이슈 (REJECTED)

| #  | 이슈 ID | 이슈          | 기각 사유        | 투표     |
|----|---------|---------------|------------------|----------|
| 1  | I-03    | 네이밍 컨벤션 | 스타일 취향 문제 | 2/3 REJECT |

### 미합의 이슈 — PM 판정 필요 (DEFER)

| #  | 이슈 ID | 이슈          | Claude    | Codex     | GLM       | 비고            |
|----|---------|---------------|-----------|-----------|-----------|-----------------|
| 1  | I-04    | 에러 처리 방식| CONFIRMED | REJECTED  | DEFER     | 과반 미달       |

### 라운드별 수렴도 추이

| 라운드   | 유효 이슈 | 합의 이슈 | 수렴도 |
|----------|-----------|-----------|--------|
| Round 1  | 5         | 1         | 20%    |
| Round 2  | 4         | 2         | 50%    |
| Round 3  | 2         | 2         | 100%   |

### 최종 GATE 판정

- 합의율: X%
- CRITICAL/HIGH 이슈 수: N건
- GATE: **PASS** | **CONDITIONAL** | **FAIL**

GATE 기준:
- PASS: CRITICAL 0건, HIGH 0건 (합의된 것만)
- CONDITIONAL: CRITICAL 0건, HIGH ≥ 1건 또는 미합의 이슈 존재
- FAIL: CRITICAL ≥ 1건 (합의된 것)

PM 액션:
- CONDITIONAL인 경우, 미합의 DEFER 이슈를 PM이 직접 판정한 후 GATE 재확정.
```

## 주의사항

1. **시간 소요**: 3라운드 × 3모델 = 최대 9회 모델 호출. 약 5-15분 소요.
2. **비용**: 단발 3-way 대비 3배 호출량. 중요 코드에만 사용 권장.
3. **병렬 실행**: 각 라운드 내 3개 모델 호출은 반드시 병렬로 실행하여 시간 단축.
4. **파일 정리**: 리뷰 종료 후 /tmp/hih_3way_hard/ 파일들을 확인하고 불필요 시 삭제.
5. **PM 판정**: DEFER 이슈는 반드시 PM이 최종 결정한다. 자동으로 CONFIRM/REJECT하지 않는다.
