# 세무 수치 SSOT (Single Source of Truth)

> **이 파일이 모든 금액·세율·기준의 원본이다.** 다른 참조 파일의 숫자가 이 파일과 다르면 **이 파일이 우선**.
> 각 값에는 `시행연도`(귀속/적용 기준)와 `검증일`(마지막으로 사람이 출처 확인한 날)과 `출처`를 단다.
> 세법은 연 1회 사이클로 바뀐다(7~8월 개정안 발표 → 12월 국회 통과 → 익년 1/1 시행). `last_verified`가 오래됐으면 스킬이 사용 시점에 재확인한다.

```yaml
meta:
  기준_귀속연도: 2025          # 2026년 5월 신고분
  last_verified: 2026-05-31
  다음_점검_권장: 2026-08-01    # 세법개정안 발표 시즌
```

## 갱신 절차 (`/hih-tax 세법 갱신`)

세법 변경 감지 알림을 받거나 `last_verified`가 6개월+ 지나면:

1. WATCHED FACTS 각 key를 **웹 재확인**(국세청/홈택스/정책브리핑 1차 출처 우선).
2. **변경된 값만 diff로 제시**: `간이과세_기준: 1억400만 → 1억XXX만 (출처: ...)`.
3. **사용자 승인 후**에만 이 파일 수정 (test-first·publish-gate: silent 자동수정 금지).
4. 수정 시 해당 행 `검증일` + `meta.last_verified` 갱신, 변경분은 아래 CHANGELOG에 기록.
5. 변경이 다른 참조 파일(business-types/tax-saving/tax-calendar) 수치에도 걸리면 함께 갱신.
