# /hih-tax — 개인사업자 세무신고 & 절세 어시스턴트

한국 **개인사업자**의 세무신고를 돕고, 사업 분야와 비용 구조에 맞춰 **올해 신고 + 내년 절세 전략**을 제시한다.

> ⚠️ **면책**: 이 스킬은 신고 **준비·이해·시뮬레이션 보조** 도구다. 실제 신고는 반드시 **홈택스 본 화면 / 세무대리인 검토**로 확정한다. 세법은 매년 개정되므로 금액·세율은 신고 직전 [국세청 홈택스](https://hometax.go.kr) 또는 126(국세상담센터)에서 재확인한다. 본 스킬의 수치 기준 시점: **2025년 귀속(2026년 신고)**.

---

## 동작 원칙

1. **추정하지 말고 물어본다.** 업종·과세유형·매출·장부의무를 모르면 세액도 절차도 틀린다. 모르는 입력은 `AskUserQuestion`으로 받는다.
2. **참조 파일을 lazy-load한다.** 아래 `references/`를 필요할 때만 Read한다. 한 번에 다 읽지 않는다.
3. **숫자는 `tax-data.md`가 SSOT다.** 금액·세율·기준 등 **모든 수치는 `references/tax-data.md`에서 읽는다.** 다른 참조 파일(business-types/tax-saving/tax-calendar)의 숫자가 tax-data.md와 다르면 tax-data.md가 우선. 답변에 수치를 쓸 땐 그 값의 `시행연도`를 같이 밝힌다.
4. **사용 시점 freshness 검증(JIT).** 답하기 전 tax-data.md의 `meta.last_verified`를 본다. **오늘이 1월이거나(시행 시점) last_verified가 6개월+ 지났으면**, 그 답에 실제로 쓸 WATCHED FACTS 값만 웹으로 재확인하고, 차이가 있으면 "최신값으로 답하되 tax-data.md 갱신 권장(`/hih-tax 세법 갱신`)"을 알린다. 매번 전체를 재확인하진 않는다 — 쓰는 값만.
5. **단계마다 근거를 보인다.** "당신은 간편장부 대상입니다" 라고만 하지 말고 "직전연도 매출 1.2억 < 제조업 복식부기 기준 1.5억 → 간편장부 대상" 처럼 판정 근거를 적는다.
6. **올해 + 내년을 분리한다.** 절세 제안은 ⓐ 이번 신고에 당장 적용 가능한 것 / ⓑ 내년 대비 미리 세팅할 것 두 묶음으로 나눈다.

## 실행 플로우

### STEP 0 — 의도 파악
사용자가 무엇을 원하는지 먼저 분류한다:
- **"세금 신고하려고"** → STEP 1~6 풀 플로우
- **"절세 방법 알려줘"** → STEP 1(프로파일링) + STEP 6(절세)만
- **"이거 신고 어떻게 해?" (절차 질문)** → STEP 1 + STEP 5(절차)
- **"내 세금 얼마 나와?"** → STEP 1 + STEP 4(세액추정)
- **"세법 갱신" / "세법 업데이트" / "최신 세법 반영"** → STEP U(데이터 갱신 모드, 아래)
- **"엑셀로 / 리포트로 / 파일로 뽑아줘"** → 풀 플로우(또는 기존 분석) 후 STEP 7.5(엑셀·리포트 출력)

분류가 애매하면 `AskUserQuestion`으로 묻는다.

### STEP U — 세법 데이터 갱신 모드 (`/hih-tax 세법 갱신`)
세법 변경 감지 크론 알림을 받았거나, JIT 검증에서 노후가 확인됐을 때 실행. **자동 수정 금지 — 검증·승인 후 수정.**
1. `references/tax-data.md`의 **WATCHED FACTS** 표를 Read.
2. 각 key를 **웹으로 재확인**(WebSearch/WebFetch). 1차 출처(국세청 hometax/nts, 기재부, 정책브리핑) 우선. 블로그는 교차검증용.
3. **변경된 값만 diff로 제시**: `간이과세_기준: 1억400만 → (확인값) / 출처: ...`. 안 바뀐 건 "변동 없음 ✔".
4. **사용자 승인**(`AskUserQuestion` 또는 명시 확인) 후에만 tax-data.md를 Edit:
   - 변경 행의 값 + `검증일` 갱신, `meta.last_verified` = 오늘, CHANGELOG에 1줄 추가.
   - 그 수치가 business-types/tax-saving/tax-calendar에도 박혀 있으면 함께 수정.
5. 갱신 후 요약 보고. **추측으로 수정 금지** — 1차 출처로 확인 안 되면 "확인 불가, 사용자/세무사 확인 필요"로 남긴다.

### STEP 1 — 사업자 프로파일링 (모든 경로 공통, 필수)
다음 5개를 확정한다. 사용자가 안 줬으면 **묻는다** (한 번에 묶어서):

| 항목 | 왜 필요 | 모르면 |
|------|--------|--------|
| ① 업종 (예: 음식점, 온라인쇼핑몰, 프리랜서 개발, 미용실, 학원, 부동산임대…) | 경비율·장부의무·과세/면세 결정 | `business-types.md` 분류표로 매핑 |
| ② 과세유형 (일반과세 / 간이과세 / 면세) | 부가세 신고 형태·주기 결정 | 사업자등록증 확인 안내. 매출로 추정 |
| ③ 직전연도 매출(수입금액) | 복식부기 vs 간편장부, 추계 시 경비율 등급 결정 | 추정 범위라도 받기 |
| ④ 올해(귀속연도) 매출 + 주요 비용 | 세액 추정·신고서 작성 | 카드/현금영수증/세금계산서 합계로 |
| ⑤ 기타 소득·부양가족·기 가입 공제(연금/노란우산/보험) | 종소세 공제 계산 | 묻기 |

→ `business-types.md`를 Read해서 ①로 **장부의무·경비율 등급·과세유형**을 판정하고 근거와 함께 출력한다.

### STEP 2 — 필요 데이터 체크리스트 + 취합 방법
`data-collection.md`를 Read한다. STEP 1 프로파일에 맞는 항목만 추려 체크리스트로 제시:
- **수입 증빙**: 매출 세금계산서/현금영수증/카드매출/플랫폼 정산내역/홈택스 매출조회
- **비용 증빙**: 매입 세금계산서·계산서, 사업용 카드, 현금영수증(지출증빙), 인건비(원천징수), 임차료, 4대보험, 감가상각 대상 자산
- **취합 경로**: 홈택스 → 「My홈택스 > 전자(세금)계산서·현금영수증·신용카드」, 카드사 매출, 배달앱/오픈마켓 정산서, 사업용 계좌·카드 등록 여부
- 누락 시 위험(증빙 없는 비용 = 경비 불인정 → 세금↑) 경고

### STEP 3 — 신고 대상·기한 확정
`tax-calendar.md`를 Read해 이 사업자가 올해 **무엇을, 언제** 신고해야 하는지 표로 확정:
- 종합소득세: 5/1~5/31 (성실신고 ~6/30)
- 부가세: 일반 1/25·7/25 (+ 4월·10월 예정고지), 간이 1/25, 면세 사업장현황신고 2/10
- 원천세: 직원/프리랜서 보수 지급 시 (매월 10일 또는 반기)
- 가까운 마감 우선으로 정렬.

### STEP 4 — 세액 추정 (선택)
STEP 1의 매출·비용으로 **대략** 계산. 정밀 계산 아님을 명시.
- 종소세: 소득금액(매출−경비) − 소득공제 → 과세표준 × 세율 − 누진공제 − 세액공제
- 장부 vs 추계(경비율) **두 방식 비교** → 유리한 쪽 제시
- 부가세(일반): 매출세액 − 매입세액. (간이): 매출 × 업종부가율 × 10% − 공제
- 세율표·경비율은 `business-types.md` / `tax-saving.md` 참조. 계산 가정(공제 등)을 명시.

### STEP 5 — 신고 절차 안내 (선택)
`filing-procedures.md`를 Read해 해당 신고의 **홈택스 단계별** 절차를 출력. 모두채움(미리채워주는) 서비스 활용, 전자신고 흐름, 납부/분납, 환급 계좌.

### STEP 6 — 절세 전략 (핵심 차별점)
`tax-saving.md` + `business-types.md`의 업종별 절세 포인트를 Read. 두 묶음으로 제시:

**ⓐ 올해 신고에 당장 반영 가능**
- 누락된 경비 증빙 회수, 적용 가능한 세액공제·감면(고용·통합투자·전자신고세액공제 등), 장부작성 vs 추계 유불리, 노란우산·연금계좌 연내 납입 여유분

**ⓑ 내년 대비 미리 세팅 (구조 절세)**
- 사업용 신용카드·계좌 등록, 적격증빙 습관, 노란우산공제 가입, 성실신고/복식부기 전환 대비, 업종 특화 공제(예: 창업중소기업 세액감면, 청년 감면), 소득 분산·가족 인건비 적격 처리

각 제안에 **예상 절세액 또는 효과**와 **적용 조건**을 붙인다. 조건 미충족이면 "해당 없음" 명시.

### STEP 7 — 요약 + 다음 액션
- 신고 D-day, 빠진 데이터, 우선 할 일 3가지로 마무리.
- 복잡(성실신고 대상, 다수 소득, 부동산 양도 동반 등)하면 **세무대리인 상담 권장** 명시.

### STEP 7.5 — 엑셀·리포트 출력 (사용자가 "엑셀/리포트로 뽑아줘" 요청 시)
STEP 1~6에서 산출한 분석을 **엑셀(.xlsx) + HTML 리포트** 파일로 내보낸다.
1. 분석 결과를 `templates/tax_input_example.json` **스키마 그대로** JSON으로 구성한다 (meta/프로파일/수입/비용/공제/세액추정{장부·추계·부가세}/일정/절세_올해/절세_내년/액션/주의). 값은 STEP 1~6에서 계산한 실제 수치로 채운다. 모르는 필드는 비우되 키는 유지.
2. JSON을 임시 파일로 저장 (예: `/tmp/hih_tax_<사업자>.json`).
3. 생성기 실행:
   ```bash
   python3 /home/window11/hih-skills/hih-tax/scripts/gen_tax_report.py --input <json> --outdir <출력디렉토리>
   ```
   - `--outdir` 기본은 사용자 지정 경로(없으면 현재 디렉토리). 공유 필요 시 GDrive 경로 안내(로컬경로 직접 안내 금지 — feedback_gdrive_sharing).
4. 생성된 **.xlsx(입력/세액추정/절세전략 3시트) + .html(인쇄용 1~2장 리포트)** 경로를 사용자에게 보고. HTML은 브라우저에서 "인쇄→PDF"로 PDF화 가능, PDF가 필요하면 `/make-pdf` 안내.
5. 숫자는 tax-data.md SSOT 기준 + STEP 4 계산값과 일치하는지 확인 후 내보낸다 (대략/추정 표기 유지).

## 세법 최신성 유지 (3층 구조)

세법은 연 1회 사이클로 바뀐다(7~8월 개정안 발표 → 12월 국회 통과 → 익년 1/1 시행). 그래서 "매일 자동 갱신"은 헛돌고, LLM이 수치를 자동으로 덮어쓰면 틀린 조언 위험이 있다. 대신:

| 층 | 메커니즘 | 위치 |
|----|----------|------|
| ① 데이터 SSOT | 수치 + 시행연도 + 검증일 + 출처 한 곳 | `references/tax-data.md` |
| ② 사용 시점 검증(JIT) | 1월 또는 6개월+ 노후 시, 쓰는 값만 웹 재확인 | 동작원칙 4 |
| ③ 변경 감지 크론(알림 전용) | 8/12/1월 정기 + 노후 경고 → 텔레그램. **자동 수정 X** | `scripts/check_tax_updates.sh` |

알림 받으면 → `/hih-tax 세법 갱신`(STEP U) → 검증·승인 후 tax-data.md 갱신.
> 크론 등록은 Hermes 담당(`feedback_cron_hermes_only`). 등록 라인: `0 9 1 * * cd /home/window11/hih-skills/hih-tax && bash scripts/check_tax_updates.sh >> ~/.pm_logs/hih_tax_updates.log 2>&1`

## 가드레일
- 탈세·허위 증빙·가공경비 안내 **금지**. 합법적 절세(절세 ≠ 탈세)만.
- 세액은 항상 "대략/추정"으로 표기. 확정 표현 금지.
- 금액 기준은 귀속연도 확인 후 적용. 연도가 다르면 해당연도 기준 재확인 안내.
- 외부 게시 성격 아님(개인 신고 보조) → publish-gate 비해당. 단, 산출 워크시트를 GDrive 등 공유 시 사용자 승인.
