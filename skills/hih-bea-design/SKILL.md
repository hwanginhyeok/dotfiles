> Purpose: consistently generate new card designs in a **dynamic (Mustache) / brand-consistent / gate-passing** state.

## Design principles (verified by external review)
1. **Design HTML is generated by Codex** (not clone+recolor — real layout design). Use cover as a few-shot for tone consistency.
2. **Reuse the scaffold**: README frontmatter + render_candidate_style.py + ai_background_gen. Do NOT build from scratch.
3. **production photo variable = `{{photo_url}}`** (injected at render_candidate_style.py:56). `{{photo_path}}` / absolute paths prohibited.
4. **accent_color is a semantic token** — no blind #D62828 substitution. brand_accent (#D62828 fixed, badge/CTA) / style_accent (free for charts) / divider (SharedPalette).

## Standard 12 slots (required for a full set)

## Generation flow (per style)
1. **Pre-learning Read**: the target cover.html (its DNA, if it exists) + photo_overlay 12 slots (the contract).
2. **Codex generation**: generate missing slots in one shot. The prompt must explicitly state — "1080×1080, {{photo_url}}, all content as Mustache variables (no hardcoding), no absolute paths, brand elements (Be:An. dot #D62828 + Be:Analogue watermark + red accent), inherit the cover tone".
3. **candidate placement**: `config/design_library/candidates/{id}/components/` + README frontmatter (copy from photo_overlay, status candidate, layouts 12 true, set photo.required true if a photo is needed).
4. **Render**: `source .env && python scripts/render_candidate_style.py {id}` (for photo slots, inject photo_url via `--content-json`; keep {{photo_url}} in the template).
5. **Gate verification**: `python scripts/verify_design_gate.py {id}` → confirm all 7 gates PASS.

## Principle for guaranteeing text contrast over photos — the scrim technique (codified 2026-06-06)

> **No matter what photo is in the background, text must always be readable (WCAG AA 4.5:1).**

### Scrim principle
- Lay a scrim (a gradient/semi-transparent panel behind the text) over **every area** where text overlaps a photo.
- Ensure text sits on a **controlled background**, not on the raw photo.
- Dark scrim + white text (recommended, most robust) or light scrim + dark text (to match the design tone).
- Opacity baseline: `linear-gradient transparent → rgba(0,0,0,0.6~0.75)` or stronger — cover the entire text box.

### Implementation pattern
```css
/* Dark scrim — bottom text area */
.photo::after {
  content: "";
  position: absolute; inset: 0;
  background: linear-gradient(180deg,
    rgba(0,0,0,0.0) 40%,
    rgba(0,0,0,0.65) 100%);
}
/* Light scrim — top eyebrow area (light-background design) */
.canvas::before {
  background: linear-gradient(to bottom,
    rgba(245,241,234,0.72) 0%,
    rgba(245,241,234,0.0) 38%, ...);
}
```

### Target slots (must include a scrim)
- `cover.html` — when {{photo_url}} is present
- `content_photo_caption.html`
- `content_photo_full.html`

### Verification
- The G8 gate (verify_design_gate.py) checks automatically: if a photo slot has no `linear-gradient(rgba)`, it WARNs.

## 8 gates (verify_design_gate.py)
| # | Gate | Criterion |
|---|------|-----------|
| 1 | No absolute paths | 0 occurrences of file:/// · /home · /mnt in url()/src=/href= |
| 2 | 12 slots complete | All standard 12 components present |
| 3 | Mustache dynamic | content body/numbers/labels = {{variable}}, 0 hardcoded samples |
| 4 | photo contract | {{photo_path}} / absolute paths prohibited; **photo slots (content_photo_caption/full + cover with photo.required=true) must have a {{photo_url}} render element in a url()/src= context** (FAIL if missing) |
| 5 | Brand elements | logo + watermark + #D62828 in all slots |
| 6 | CSS variables | var(--) recommended (warn) |
| 7 | Renderable | samples/*.png >5KB |
| 8 | Text-over-photo contrast (scrim) | WARN if photo slots (cover/photo_caption/photo_full) have no linear-gradient rgba overlay |

→ FAIL must be 0 to be accepted as a candidate. **Promotion (templates+format_map) is publish-gate** (after user visual approval).

## Multi-agent bulk generation (lessons)

- When dispatching, reference the shared contract file + style groups of 3 each.
