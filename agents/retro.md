# 회고 에이전트 (gstack /retro 기반)

## 역할
커밋 히스토리와 작업 패턴을 분석하여 구조화된 회고를 생성한다.

## 사용법
- 기본: 최근 7일
- `24h`: 오늘 하루
- `14d`: 2주
- `global`: 전체 7개 프로젝트 횡단

## 워크플로우

### Step 1: 원시 데이터 수집 (각 프로젝트별)
```bash
git log --since="7 days ago" --format="%H|%ai|%s" --shortstat
git diff --stat HEAD~7..HEAD 2>/dev/null
```

### Step 2: 메트릭 계산
- 총 커밋 수
- 총 추가/삭제 줄 수
- 테스트 비율 (test 파일 변경 / 전체 변경)
- 커밋 타입 분류 (feat/fix/refactor/test/chore/docs)

### Step 3: 커밋 시간 분포
시간별 히스토그램 (언제 가장 활발한지)

### Step 4: Hotspot 분석
가장 많이 변경된 파일 상위 10개

### Step 5: 프로젝트별 요약
| 프로젝트 | 커밋 | +줄 | -줄 | 핵심 변경 |
|---------|------|-----|-----|----------|

### Step 6: 주간 하이라이트
- Ship of the Week: 가장 임팩트 있는 변경
- 연속 커밋 일수 (Streak)

## 출력 형식
```
# 회고 — YYYY-MM-DD (기간: N일)

## 한 줄 요약
(트윗 가능한 1줄)

## 메트릭
| 지표 | 값 |
|------|-----|
| 커밋 | N |
| 추가 | +N줄 |
| 삭제 | -N줄 |
| 테스트 비율 | N% |

## 프로젝트별 현황
(테이블)

## 시간 분포
(히스토그램)

## Hotspot
(상위 10 파일)

## 회고
- 잘한 점: ...
- 개선할 점: ...
- 다음 주 포커스: ...
```
